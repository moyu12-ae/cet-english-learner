# 四六级英语深度学习技能

## 核心理念

基于费曼学习法：以"做"和"用"去促进"学"、检验"学"。不是孤立背单词，而是**在真实语境中学习词汇和表达**。

本技能的角色定位：**学习内容的加工者和学习路径的引导者**，而非学习效果的评判者。准备好"食材"（提取词汇、拆解句式）和"菜谱"（精读讲解、复述引导），但"做菜"（主动输出）由用户自己完成。

## 触发场景

用户在提供英语文本后，明确表示要进行学习。常见说法：
- "帮我精读这篇文章"
- "分析这篇英语新闻"
- "帮我从这个字幕学英语"
- "生成墨墨记忆卡卡片"
- "四六级词汇提取"

## 工作流

### 阶段 0：语料识别与预处理

**步骤 1**：识别语料类型
- 如果有标题、段落结构、标点符号完整 → **文章模式**
- 如果全是对话片段、无标点符号（或极少标点）、多行短句 → **字幕模式**
- 如果用户明确说了来源 → 按用户说明判断

**步骤 2**：预处理

文章模式：
- 清理 HTML 残留、多余空行
- 保留标题和来源信息
- 按自然段落分段

字幕模式：
- 用户提供的字幕已经是去掉时间戳的纯文本（无标点符号）
- 按语义自动分句，将碎片行合并为完整对话段落
- 按对话轮次或场景分段

**步骤 3**：用 AskUserQuestion 工具确认语料类型并收集配置偏好。

你已经完成了语料自动识别和分段处理，现在用两轮 AskUserQuestion 让用户点选确认和配置。每个选项的最后一行为推荐选项，标注「(推荐)」。

重要提醒：调用 AskUserQuestion 时，不要传入 `preview` 字段（它不是必填项，传空会报错）。只传 `header`、`question`、`options`（含 `label` 和 `description`）、`multiSelect`。

**步骤 4**：初始化输出目录（调用脚本）

根据文章标题生成目录名：
- 提取英文标题的前 3-5 个核心单词，转为小写，空格替换为连字符 `-`
- 去掉标点符号（如逗号、冒号），保留字母和数字
- 末尾加 `-learning`
- 示例：`Lupin III Composer Yuji Ohno Dies at 84` → `lupin-iii-composer-learning`
- 字幕模式：用视频/剧集名称简写，或根据内容主题命名（如 `restaurant-dialogue-learning`）
- 如果无法自动提取标题（如用户直接粘贴文本），询问用户希望用什么目录名

然后调用初始化脚本。你需要定位到 skill 目录下的 `scripts/init_learning_dir.py`：
- skill 的 SKILL.md 路径已知，`scripts/` 是其同级子目录
- 用 `python3 <skill目录>/scripts/init_learning_dir.py --source <原始语料路径> --output-dir <目录名>` 执行

脚本会自动完成：
1. 在当前工作目录创建 `{目录名}/` 文件夹
2. 将原始语料文件复制进去，命名为 `source.txt`
3. 创建 5 个空文件：`词汇表.csv`、`关键句式.md`、`精读笔记.md`、`费曼练习.md`、`墨墨卡片.txt`

调用示例：
```bash
python3 /path/to/cet-english-learner/scripts/init_learning_dir.py \
  --source ./evals/files/lupin-article.txt \
  --output-dir lupin-iii-composer-learning
```

执行成功后，向用户展示创建的目录结构和文件列表。后续各阶段只需往对应文件写入内容即可。

### 阶段 1：提取与分析

**A. 四六级词汇表**

逐段扫描，提取四六级词汇。输出格式：

```
## 第 X 段词汇

| 单词 | 级别 | 词性 | 释义 | 原文句 |
|------|------|------|------|--------|
| composer | CET4 | n. | 作曲家 | "...as a jazz pianist and composer" |
```

提取规则：
- 对照四六级词表提取（需要你认知范围内的词表知识）
- 标注级别：CET4 / CET6
- 超纲但值得学的词（如专业术语、高频学术词）：单独标注为"超纲推荐"
- 专有名词（人名、地名、作品名）：不纳入词汇学习，留到阶段2文化注中解释
- 一词多义：标注原文中的具体含义，必要时备注其他常见含义
- 短语/搭配：如果包含四六级词汇的重要搭配（如 "involvement with"），单独列在词汇表末尾

**写入文件**：`词汇表.csv` 已由初始化脚本创建，直接写入内容。CSV 格式：
- 表头：`段落,单词,级别,词性,释义,原文句`
- 原文句中的逗号用中文逗号 `，` 替代，避免 CSV 分隔符冲突
- 短语/搭配行在「单词」列标注为 `[搭配] xxx`

**B. 关键句式清单**

从全文中选出值得背记的句子。选择标准：
- 包含 2+ 个四六级词汇
- 使用了典型英语语法结构（定语从句、虚拟语气、强调句、倒装、非谓语等）
- 对理解文章核心信息关键

输出格式：
```
## 关键句式

1. **[句子原文]**
   - 句式：非限定性定语从句 + 时间状语
   - 含四六级词：soundtrack (CET4), iconic (CET6), persisted (CET6), arrangements (CET4)
   - 适用场景：描述作品/人物的持续影响力
```

**C. 段落结构概览**

每段用一句中文概括，帮助建立文章框架。

**写入文件**：将「段落结构概览」和「关键句式清单」合并写入 `{输出目录}/关键句式.md`：
```
# {文章标题} — 关键句式

## 段落结构概览
（每段一句中文概括）

## 关键句式
（所有关键句式，含句式分析和适用场景）
```

### 阶段 2：逐段精读

对每一段，按以下格式输出。注意：要逐段输出，让用户有时间消化每段内容后再看下一段。

```markdown
---

## 第 X 段精读

**原文**：
> <英文原文>

**逐句拆解**：
1. <原文句1>
   - 词汇：word1 = 释义, word2 = 释义
   - 句式：<语法结构分析>
   - 翻译：<中文翻译>

2. <原文句2>
   ...（同上）

**段落翻译**（流畅中文）：
<整段中文翻译>

**文化/背景注**：
- <文化知识点，帮助理解文化背景>
- <如果有专有名词，在此解释>
```

精读要点：
- 逐句拆解不等于逐词翻译。重点分析句式结构和关键表达
- 文化注要服务于"既学语言又学文化"的目标。对于动漫新闻，解释相关作品背景、业界常识
- 对于字幕：额外分析口语表达、语气、社交语境。比如"I could ask you the same thing"表达了什么潜台词
- 翻译应该通顺自然，不是直译

**写入文件**：将所有段落的精读内容汇总写入 `{输出目录}/精读笔记.md`。使用 Markdown 格式，按段落编号组织。每完成一段就追加写入，这样即使后续段落出问题，已完成的段也已保存。

### 阶段 3：费曼复述引导

这是费曼学习法的核心环节。在用户理解了段落内容后，引导其**主动输出**。

**3A. 理解检测问题**

为每段生成 2-3 个需要用英文回答的问题：

```markdown
## 费曼复述引导 — 第 X 段

请用英文回答以下问题：

1. <针对段落核心信息的问题>
2. <针对细节的问题>
3. <可选：需要整合多句信息的问题>

---

**参考答案**：
1. <英文参考答案>
2. <英文参考答案>
3. <英文参考答案>
```

**3B. 段落复述任务**

```
## 段落复述 — 第 X 段

用 2-3 句英文复述本段大意，不要看原文。

提示词：keyword1, keyword2, keyword3

---
**参考复述**：
<2-3 句英文复述>
```

**3C. 翻译还原任务**（如果用户配置了"完整"深度的费曼复述）

```
## 翻译还原 — 第 X 段

看中文翻译，尝试还原英文原文：

<本段流畅中文翻译>

---
**英文原文**：
<英文原文>
```

**写入文件**：将所有费曼复述内容汇总写入 `{输出目录}/费曼练习.md`。结构如下：
```
# {文章标题} — 费曼复述练习

## 理解检测问题
（按段落组织，每题后附参考答案）

## 段落复述任务
（按段落组织，附提示词和参考复述）

## 翻译还原
（如果配置了完整深度才包含此部分）
```
注意：参考答案和参考复述用折叠标记（`<details>`）包裹，方便用户先自行练习再对照。

### 阶段 4：生成墨墨记忆卡导入文件

这是输出交付物。生成一个纯文本文件，可直接导入墨墨记忆卡。

**重要格式规则**：
- 纯文本，不支持 Markdown
- 章节用 `###章节名` 标记
- 卡片之间用**空行**分割
- 答案线与题目之间用 `---` 分割
- 文字挖空格式：`[F##文字内容]`
- 多选题格式见下方

**卡片类型与模板**：

**4A. 词汇挖空卡**（核心卡片）

用原文段落的句子挖掉目标词汇。每个挖空一张卡片。

```
###文章标题 - 第X段 词汇挖空

<包含一个 [F##目标词] 的句子，保留足够上下文>
---
<该词的释义>
```

挖空策略：
- 默认（标准型）：每段挖掉所有四六级词汇，每个词一张卡片
- 如果用户指定了宽松型：每段选 3-5 个最核心的四六级词汇
- 如果用户指定了密集型：在标准型基础上额外挖掉关键连接词/功能词

**4B. 句子背记卡**

```
###文章标题 - 关键句式

<完整句子原文>
---
<句子翻译 或 使用场景说明>
```

**4C. 理解检测多选题**

```
在本段中，"<单词>"最接近的意思是？
[Choice##
* 正确释义
- 错误选项1
- 错误选项2
- 错误选项3]
```

多选题规则：
- 题目和 `[Choice##` 之间无空行
- `*` 开头的是正确答案
- `-` 开头的是错误答案
- `[Choice##` 和 `]` 包裹所有选项

**4D. 翻译还原卡**（进阶可选）

```
<本段中文翻译>
---
<英文原文段落>
```

**4E. 字幕专用：场景还原卡**

```
<角色标识>："<部分对话>"
<角色标识>："[F##挖空的回答]"

（题目区说明语气或场景）
```

**写入文件**：将所有卡片按章节组织，写入 `{输出目录}/墨墨卡片.txt`。文件内部结构：
```
###词汇挖空卡
（所有词汇挖空卡，按段落顺序）

###关键句式卡
（所有句子背记卡）

###理解检测
（所有多选题）

###翻译还原
（如果有 4D 卡片）

###场景还原
（字幕模式专用）
```
告知用户导入方法：墨墨记忆卡 → 导入 → 选择「墨墨卡片.txt」即可。

### 阶段 5：完成汇总

所有阶段完成后，汇总告知用户输出文件的完整清单：

```markdown
## 学习资料已生成

所有文件保存在 `{输出目录}/` 中：

| 文件 | 内容 | 用途 |
|------|------|------|
| `source.txt` | 原始英语语料 | 随时回顾原文 |
| `词汇表.csv` | 四六级词汇表 | 导入 Excel/Notion 复习 |
| `关键句式.md` | 段落概览 + 关键句式 | 掌握句型和语法 |
| `精读笔记.md` | 逐段拆解 + 翻译 + 文化注 | 深度理解全文 |
| `费曼练习.md` | 理解问题 + 复述任务 + 翻译还原 | 主动输出练习 |
| `墨墨卡片.txt` | 墨墨记忆卡导入文件 | 导入墨墨背单词 SRS 复习 |

**复习建议**：
- **今天**：阅读精读笔记 + 完成费曼练习中的所有问题
- **明天起**：将墨墨卡片.txt 导入墨墨记忆卡，按 SRS 算法每日复习
- **一周后**：重读原文（source.txt），检验理解流畅度
```

## 两种语料处理差异总结

| 维度 | 文章模式 | 字幕模式 |
|------|----------|----------|
| 分段方式 | 自然段落 | 对话轮次/场景 |
| 精读重点 | 书面句式、长难句 | 口语表达、省略句、俚语 |
| 文化注 | 背景知识、事件脉络 | 语气分析、社交语境 |
| 卡片类型 | A+B+C(+D) | A+E(场景还原) |
| 附加提醒 | 无 | 建议回看视频盲听 |

## 输出文件总览

所有学习产物统一输出到 `{title-slug}-learning/` 目录下。目录和空文件由 `scripts/init_learning_dir.py` 在阶段 0 一次性创建，之后各阶段只需往对应文件写入内容。

### 目录结构

```
{title-slug}-learning/
├── source.txt           # 原始英语语料副本（脚本自动复制）
├── 词汇表.csv           # 阶段1A：四六级词汇表，CSV 格式可导入其他工具
├── 关键句式.md           # 阶段1B+1C：段落结构概览 + 关键句式分析
├── 精读笔记.md           # 阶段2：逐段精读，含拆解、翻译、文化注
├── 费曼练习.md           # 阶段3：理解检测问题 + 复述任务 + 翻译还原
└── 墨墨卡片.txt          # 阶段4：墨墨记忆卡标准导入格式
```

### 写入策略

- 阶段 0 调用 `init_learning_dir.py` 后，6 个文件（含 source.txt）均已就位
- **立即写入**：每完成一个阶段的全部内容后，一次性写入对应文件
- **追加写入**：精读笔记和费曼练习内容较多，每完成一段就追加写入，防止上下文丢失导致内容丢失
- **墨墨卡片**：在阶段 4 一次性生成全部卡片后写入

### 完成时汇总

阶段 5 中向用户展示所有输出文件的路径和用途表格（见阶段 5 模板），让用户一目了然。

## 关键边界处理

### 难度控制
- 如果一篇文章的四六级词汇密度过高（>15% 的单词是四六级词汇），提醒用户："这篇文章的四六级词汇密度较高（约 X%），可能需要较多时间消化。"
- 如果几乎没有四六级词汇（<2%），提醒："这篇文章的四六级词汇较少，可能更适合泛读而非精读。"

### 卡片数量控制
- 单段挖空卡不超过 12 张。超过时提醒用户是否拆分或减少
- 整篇文章的卡片总数建议在 30-60 张之间

### 墨墨格式注意事项
- 空行是卡片分割符，段落中如果有空行（如诗歌、对话分段），用 `[---]` 占位替代
- 单张卡片内容不超过 150 词，超过的段落拆分为多张卡片
- `###` 只在章节标题使用，正文中避免

### 字幕特有的语义分句
分句判断依据：
- 主语+谓语的完整性
- 对话的自然停顿
- 上下文语义连贯性
- 如果一句话被切成了多行碎片（常见于字幕），合并为完整句子再处理
